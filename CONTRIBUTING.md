# 기여 가이드

`activist-scout` 에 기여를 환영합니다. 다음 영역이 특히 도움이 됩니다.

## 환영하는 기여

### 1. 도메인 지식 갱신

- `src/activist_scout/domain.py` 의 행동주의 펀드 명단 추가/갱신
- 동종업계 비교 대상 종목 매핑 정확도 개선
- 산업별 행동주의 환경 정보 보강

### 2. 데이터 소스 확장

- 한국 부동산 공시 지가 데이터 통합
- 외국 행동주의 펀드의 한국 진입 모니터링
- 추가 공시 항목 활용 (회계 감사인 변경, 공정거래위원회 제재 등)

### 3. 자동 테스트 보강

- `tests/` 에 새 테스트 케이스 추가
- 특히 도메인 규칙의 가장자리 사례

### 4. 문서·예시

- 사용 사례 추가 (`examples/` 폴더)
- 한국 행동주의 캠페인 케이스 스터디 추가

## 기여 절차

1. 저장소를 본인 계정으로 복사 (Fork)
2. 새 브랜치 생성: `git checkout -b feature/원하는-기능`
3. 코드 변경 + 테스트 작성
4. 코드 검사 통과 확인: `ruff check src tests`
5. 자동 테스트 통과 확인: `pytest tests/`
6. Commit + Push
7. Pull Request 생성

## 코드 스타일

- Python 3.11 이상
- `ruff check` 통과
- 모든 외부 공개 함수에 짧은 한국어 또는 영어 설명
- 도메인 규칙 변경 시 [STORY.md](STORY.md) 또는 [CHANGELOG.md](CHANGELOG.md) 에
  *왜* 변경했는지 기록

## 절대 받지 않는 변경

다음 변경 제안은 그 어떤 형태로도 받지 않습니다:

1. **"⚠️ 사람 검증 필수 항목" 섹션 제거** — 이건 시스템의 정직성 핵심입니다.
   자동화가 못 하는 것을 분명히 표시하는 것이 가장 중요한 안전 장치입니다.
2. **과거 데이터로 미래 수익을 예측한다는 주장** — 한국 상법 개정으로 환경이
   바뀌었기 때문에, 그 이전 데이터로 미래를 단정해서는 안 됩니다.
3. **투자 권유성 텍스트** ("이 종목 사세요" 같은 표현)
4. **`.env` 파일 또는 그 밖의 비밀 정보 commit**
